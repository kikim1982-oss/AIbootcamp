# 📜 [Server+DB] 냉장고 재료 & 레시피 관리앱

> 🎯 **Server + DB (Supabase)** 를 조합해서 냉장고 재료와 레시피를 저장/관리하는 앱을 만드세요!

---

## 미션

1. 사용자가 **냉장고에 있는 재료를 등록/삭제**할 수 있게 하세요
2. 사용자가 **레시피를 직접 작성하고 저장**할 수 있게 하세요 (요리명, 재료, 조리법)
3. 저장된 재료와 레시피를 **목록으로 조회**할 수 있게 하세요
4. 모든 데이터는 **DB (Supabase)에 저장**하세요

---

## 핵심 구조

```
[재료 등록/삭제] → [Server] → DB에 저장 → [재료 목록 조회]
[레시피 작성] → [Server] → DB에 저장 → [레시피 목록 조회]
```

- **DB 역할:** 재료 테이블 + 레시피 테이블 저장/관리 (CRUD)
- **Server 역할:** 재료 CRUD API + 레시피 CRUD API

---

## 제출물

- GitHub 저장소 링크 (코드 포함)
- 동작 스크린샷 (최소 1장)
- 에이전트와의 대화 스크린샷 (최소 1장)

---

## 포인트 획득 기준

| 항목 | 포인트 | 기준 |
|---|---|---|
| 기본 완료 | 10pt | 제출물을 기한 내 제출 |
| 에이전트 활용 | 5pt | 에이전트와 2회 이상 대화하며 결과물을 개선한 흔적 |
| 창의성 | 5pt | 자기만의 개성이 담긴 결과물 (카테고리 분류, 유통기한 관리, 검색 기능 등) |
| 공유 보너스 | 5pt | 단톡방에 결과물을 공유하고 다른 수강생 결과물에 리액션 |

| 달성 수준 | 획득 포인트 |
|---|---|
| 기본 제출 | 10pt |
| 기본 + 활용/창의성 | 15~20pt |
| 올클리어 (전항목 달성) | **25pt** |
