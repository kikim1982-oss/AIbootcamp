# 📜 Quest 1 — 명령 실행 결과 정리

> Claude Code + Notion MCP 연동을 통해 실제 워크스페이스에서 실행한 명령 4개와 한 줄 활용 설명

---

## 🎯 명령 1 — 카드 형태 일정 조회

**스크린샷:** `screenshots/20260430_143037.png`

**입력 명령:**
> "5월 데이트 일정 4건 장소·시간만 카드 형태로 출력해줘"

**실행 결과:**
일정 DB에서 5월 데이트 일정 5건을 조회해 카드 UI로 시각화 (제목·날짜·시간·장소·메모 한눈에)

**📌 한 줄 활용:**
> 캘린더를 안 열고 채팅창에서 **특정 카테고리 일정만 빠르게 훑어보고 싶을 때** 유용 — "이번 주 미팅만", "다음 달 운동만" 식으로 응용 가능

---

## 🎯 명령 2 — 페이지 자동 업데이트 + 중요도 분류

**스크린샷:** `screenshots/20260430_143338.png` (Claude 응답) · `screenshots/20260430_145138.png` (Notion 페이지 결과)

**입력 명령:**
> "이 페이지에 다음주 1주간 스케줄 요약해주고 중요도 체크해줘"

**실행 결과:**
스케줄 페이지 본문에 5/4~5/10 일정 11건을 표로 추가하고, 각 항목에 🔴 HIGH / 🟡 MED / 🟢 LOW 중요도 자동 부여 + TOP 3 우선순위 추출

**📌 한 줄 활용:**
> **매주 일요일 저녁 한 주 계획 짤 때** 핵심 — Claude가 일정 DB에서 다음 주 항목만 골라 페이지에 자동 정리 → 중요도까지 매겨주니 의사결정 시간 단축

---

## 🎯 명령 3 — 데이터 기반 랭킹 분석

**스크린샷:** `screenshots/20260430_145746.png`

**입력 명령:**
> "노트 태그 확인해서 중요도 순서 랭킹 만들어서 테이블로 보여줘"

**실행 결과:**
노트 DB 18건에 대해 태그(중요/TODO/진행중)·즐겨찾기·상태를 가중치로 점수화하고 18위까지 전체 랭킹 테이블 출력

**📌 한 줄 활용:**
> **쌓인 노트가 많아서 뭐부터 봐야 할지 모를 때** 유용 — 단순 검색이 아니라 "지금 가장 중요한 것" 기준으로 노트를 자동 정렬해서 우선순위 파악

---

## 🎯 명령 4 — DB 스키마 변경 + 일괄 업데이트

**스크린샷:** `screenshots/20260430_150107.png` (Claude 응답) · `screenshots/20260430_150125.png` (Notion DB 결과)

**입력 명령:**
> "이 중요도 랭킹 결과를 DB페이지에 필드하나 추가해서 첨부해줘"

**실행 결과:**
노트 DB에 `중요도` SELECT 컬럼(5단계: HIGH ~ ARCHIVE) 추가 + 18개 노트에 분석 결과 일괄 입력 (병렬 처리)

**📌 한 줄 활용:**
> **분석 결과가 일회성으로 끝나지 않게 DB 스키마에 영구 저장**할 때 핵심 — Claude가 컬럼 신설부터 18개 row 일괄 update까지 한 번의 명령으로 처리, 사람이 손으로 하면 30분 걸릴 작업이 30초

---

## 💡 보너스 — 자동화 아이디어

**"매주 일요일 저녁 → 이번주 완료 일정 + 다음주 해야할 일을 스케줄 페이지에 자동 업데이트"**

→ `/schedule` 슬래시 커맨드로 cron routine 등록 가능 (매주 일요일 21:00 KST)
→ 사전 조건: claude.ai에 Notion 커넥터 연결 (Max 플랜)
→ 등록 후엔 사용자가 잊고 있어도 일요일 밤에 알아서 페이지 갱신 → **진정한 AI 비서 완성**

---

## 📊 4개 명령이 보여주는 MCP의 4가지 활용 패턴

| 패턴 | 예시 명령 | 본질 |
| --- | --- | --- |
| **조회 + 시각화** | 명령 1 | DB → 사람이 읽기 좋은 형태 |
| **분석 + 페이지 작성** | 명령 2 | 데이터 가공 + 결과를 다른 페이지에 기록 |
| **다중 객체 분석** | 명령 3 | 여러 row의 메타데이터 종합 평가 |
| **스키마 변경 + 일괄 쓰기** | 명령 4 | 분석 결과의 영속화 |

> 4주차까지는 **내가 만든 DB**만 다뤘지만, MCP를 쓰면 **실제 사용 중인 Notion 워크스페이스**에 동일한 패턴을 그대로 적용할 수 있다 — 이게 핵심.
